# Core API overview

The full API reference is split into three sections, mirroring the shape of the SDK itself: a component you render, the retained state object behind it (when you need one), and the provider-independent geometry/camera types both are built from.

## Components

Each overlay component accepts either convenience props (for small or static overlays) or a `state` prop (for large or frequently-updated ones) — see [Overlays](/guides/overlays/) for when to use which.

| Component | Reference |
| --- | --- |
| Map view (shared contract across all providers) | [MapView component](/components/mapview/) |
| `Marker` / `Markers` | [Marker](/components/marker/) |
| `Circle` | [Circle](/components/circle/) |
| `Polyline` | [Polyline](/components/polyline/) |
| `Polygon` | [Polygon](/components/polygon/) |
| `GroundImage` | [GroundImage](/components/groundimage/) |
| `RasterLayer` | [RasterLayer](/components/rasterlayer/) |
| `InfoBubble` / `InfoBubbleAtPosition` / `InfoBubbleCustom` | [InfoBubble](/components/infobubble/) |
