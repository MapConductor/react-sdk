# GeoPoint

`GeoPoint` represents a geographic coordinate with latitude, longitude, and an optional altitude. It's the type every overlay position and `MapCameraPosition.position` is built from.

## Interface

```ts
interface GeoPoint {
  latitude: number;        // -90 to 90
  longitude: number;       // -180 to 180
  altitude?: number | null; // meters

  isValid(): boolean;
  normalize(): GeoPoint;
  wrap(): GeoPoint;
  equals(other: GeoPoint, tolerance?: number): boolean;
  hashCode(): number;
  toUrlValue(precision?: number): string;
}
```

`GeoPointInterface` is the plain `{ latitude, longitude, altitude? }` shape without the methods — accepted anywhere a full `GeoPoint` isn't required (e.g. `createGeoRectBounds`'s `extend()`), and normalized to a real `GeoPoint` internally via `fromGeoPoint()`.

## Creating a point

```ts
GeoPoint.from({ latitude, longitude, altitude? })
GeoPoint.fromLatLng(latitude, longitude, altitude?)
GeoPoint.fromLngLat(longitude, latitude, altitude?)

createGeoPoint({ latitude, longitude, altitude? })
fromLatLng({ latitude, longitude, altitude? })   // alias: fromLatLong
fromLngLat({ longitude, latitude, altitude? })   // alias: fromLongLat
fromGeoPoint(point)  // normalizes any GeoPointInterface into a full GeoPoint
```

```tsx
const tokyo = GeoPoint.from({ latitude: 35.6812, longitude: 139.7671 });

<Marker position={tokyo} />
```

## Validation and normalization

`isValid()` checks both ranges are satisfied (`-90 ≤ latitude ≤ 90` and `-180 ≤ longitude ≤ 180`).

Two methods bring out-of-range coordinates back into range, with different semantics:

- **`normalize()`** clamps latitude to `[-90, 90]` and wraps longitude modulo `360` into `[-180, 180]`. Altitude defaults to `0` if unset. This does not model crossing a pole — it just clamps.
- **`wrap()`** properly wraps a point that has gone past a pole, as if walking over the top of the globe: past `90°` or `-90°` latitude, it reflects the latitude back and flips longitude by `180°`, then normalizes longitude. Use this over `normalize()` whenever a computed point (e.g. an offset or interpolation result) might have legitimately crossed a pole.

```ts
GeoPoint.from({ latitude: 95, longitude: 10 }).wrap();
// → { latitude: 85, longitude: -170 }
```

## Comparing points

`equals(other, tolerance = 1e-7)` compares latitude, longitude, and altitude within a tolerance (`1e-7°` ≈ 1.1cm) to absorb floating-point error — don't compare `GeoPoint`s with `===` or manual field checks.

## URL serialization

`toUrlValue(precision = 6)` formats `"latitude,longitude"` (6 digits ≈ 11cm precision), truncated (not rounded) to the given precision — useful for building deep links into Google Maps, a share URL, or any query-string-based mapping service.

```ts
tokyo.toUrlValue();     // "35.681200,139.767100"
tokyo.toUrlValue(3);    // "35.681,139.767"
```
