# GroundImage

`GroundImage` drapes an image over a `GeoRectBounds` rectangle — a floor plan, historical map overlay, or aerial photo georeferenced to real-world coordinates.

## Convenience props

```tsx
<GroundImage
  bounds={GeoRectBounds}
  imageUrl={string}
  id?: string | null
  opacity?: number     // default 0.5
  tileSize?: number    // default 256
  extra?: Serializable | null
  onClick?: (state: GroundImageState) => void
/>
```

## Retained state

```tsx
<GroundImage state={groundImageState} />
```

`state` is created with [`createGroundImageState()`](/states/groundimage-state/). Note the raw factory defaults `opacity` to `1.0`, while this convenience-prop form defaults it to `0.5` — pass an explicit value if the distinction matters to you.

## Example

```tsx
<GroundImage
  bounds={floorPlanBounds}
  imageUrl="/floor-plans/level-3.png"
  opacity={0.7}
/>
```

## Toggling visibility

There's no `visible` flag on `GroundImageState` (unlike `RasterLayerState`) — animate/toggle via `opacity`, or mount/unmount the component.

## Events

`onClick` receives the `GroundImageState`; the clicked geographic point may be `null` on providers that can't resolve an exact click position for this overlay type.
